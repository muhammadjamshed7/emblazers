### Authentication Flow
```
Login → POST /api/auth/login (admin) | /api/teacher/login | /api/student-portal/login
     → JWT issued (3-day expiry)
     → Stored in localStorage:
         Admin:   emblazers_token
         Teacher: teacher_token
         Student: student_token
     → All requests: Authorization: Bearer <token>
```

### Module Isolation (Security)
Every API request goes through `moduleAuthMiddleware` which:
1. Extracts JWT from Authorization header
2. Verifies token
3. Maps the route to allowed modules using `routeToModulesMap`
4. Blocks (403) if user's module is not in the allowed list
5. Denies unmapped routes by default (fail-closed)

### Key Data Relationships
- `studentId` links: Fee, Attendance, Library, Transport, Hostel, Curriculum
- `staffId` links: Payroll, Attendance, Timetable, Curriculum (teacher)
- Fee payment → auto-creates LedgerEntry + FinanceVoucher in Finance
- Payroll marked Paid → auto-creates FinanceVoucher in Finance

## YOUR CODING RULES

### General
- **Simple over clever.** If a junior dev can't read it in 30 seconds, simplify it.
- **No unnecessary abstractions.** Don't create a helper function unless it's used 3+ times.
- **No dead code.** Remove anything not being used.
- **One responsibility per function.** A route handler does one thing — validate, query, respond.
- **Fail loudly.** Errors should surface clearly, not be swallowed silently.

### Backend (Express + Mongoose)
- Route handlers follow this pattern exactly:
  ```typescript
  // 1. Validate input (Zod)
  // 2. DB operation
  // 3. Return response
  // No extra layers unless truly needed
  ```
- Use `async/await` — no `.then()` chains
- Always use `try/catch` in route handlers
- Return consistent error shape: `{ error: "MESSAGE" }`
- Return consistent success shape: `{ data: ... }` or just the object
- Use MongoDB transactions ONLY for multi-document writes that must be atomic
- Use `lean()` on Mongoose queries when you don't need Mongoose methods (faster)
- Add indexes for fields used in `.find()` queries frequently

### Frontend (React + TypeScript)
- Use `useQuery` for GET, `useMutation` for POST/PATCH/DELETE
- Invalidate queries after mutations: `queryClient.invalidateQueries({ queryKey: [...] })`
- Use `shadcn/ui` components — don't build custom UI from scratch for standard elements
- Keep components under 200 lines — split if larger
- No inline styles — use Tailwind classes
- Loading states: show `<Skeleton>` components, not spinners for everything
- Error states: show a clear message with a retry button
- Forms: use `react-hook-form` + Zod resolver for validation

### TypeScript
- Define types in `shared/schema.ts` — both server and client import from there
- No `any` types — use `unknown` and narrow it
- Prefer `interface` for objects, `type` for unions/intersections
- Export types alongside the Zod schemas they come from

### Simplification Rules
When you see these patterns, **replace them with the simpler version:**

| Complex Pattern | Simple Replacement |
|----------------|-------------------|
| `if (x !== null && x !== undefined)` | `if (x != null)` |
| `.find().then(r => r).catch(e => e)` | `await .find()` in try/catch |
| Separate file for a 3-line utility | Inline it where used |
| `useState` + `useEffect` for derived data | `useMemo` or compute inline |
| Giant switch statement for routes | Route-specific handler files |
| Nested ternaries deeper than 2 levels | Early returns or if/else |
| Class component | Functional component with hooks |

## KNOWN ISSUES TO FIX (Priority Order)

### 🔴 Critical (Fix First)
1. **No MongoDB transaction on payment** — if Finance write fails after Payment is saved, data is corrupted. Wrap POST /api/payments in a session transaction.
2. **Duplicate payments possible** — add `idempotencyKey` field to Payment model + check before creating.
3. **Rate limiting missing** — add `express-rate-limit` to all 3 login endpoints (max 10 attempts / 15 min).

### 🟠 Important
4. **JWT in localStorage** — move to `httpOnly` cookies to prevent XSS token theft.
5. **Quiz timer no auto-submit** — use `navigator.sendBeacon()` on `beforeunload` to submit quiz answers even if tab closes.
6. **Payment delete doesn't reverse Finance entries** — store `ledgerEntryId` + `financeVoucherId` on Payment, reverse them on delete.
7. **Short answer grading UX** — add "Needs Grading" filter tab in teacher quiz results page.

### 🟡 Nice to Have
8. **Route permission map** — move `routeToModulesMap` from middleware to `server/config/route-permissions.ts`.
9. **Bulk student account creation** — return `{ created, alreadyExisted, failed }` instead of just a count.
10. **Teacher login error** — return `NO_ASSIGNMENT` error code when teacher has no active assignments.
11. **Slim JWT payload** — remove name/class/section from token, fetch from DB when needed.
12. **Cross-module student projection** — non-student modules should get slimmed student data (no CNIC/bForm).
