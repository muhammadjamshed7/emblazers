PART 2 — BACKEND API ROUTES
A. Curriculum Admin Routes (existing JWT, add new ones)
GET  /api/curriculum/staff-teachers
     → Return all staff from existing `staff` collection where designation includes "Teacher"

GET  /api/curriculum/teacher-assignments
     → Return all teacherAssignments documents

POST /api/curriculum/teacher-assignments
     Body: { staffId, staffName, staffEmail, className, section, subject }
     → Create new assignment

DELETE /api/curriculum/teacher-assignments/:id
     → Delete assignment

GET  /api/curriculum/student-accounts
     → Return all studentPortalAccounts with isActive status

POST /api/curriculum/student-accounts/create
     Body: { studentId } OR { className, section } for bulk
     → Find student(s) in existing `students` collection
     → Create studentPortalAccount with:
         passwordHash = bcrypt(student dateOfBirth in DDMMYYYY format)
         isFirstLogin = true
     → Return created accounts count

POST /api/curriculum/student-accounts/reset-password/:studentId
     → Reset password back to DOB-based default
     → Set isFirstLogin = true again

GET  /api/curriculum/quiz-overview
     → All quizzes across all teachers with attempt counts

B. Teacher Auth Routes
POST /api/teacher/login
     Body: { staffEmail, password }
     → Check staff collection for staffEmail
     → Check teacherAssignments — if no assignment found, reject login with message "You have not been assigned any class yet. Contact admin."
     → Default password = staffId value (e.g. "STF-2024-001")
     → On success return JWT: { staffId, staffEmail, staffName, role: "teacher" }
     → Also return their assignments array

POST /api/teacher/change-password
     Auth: Teacher JWT required
     Body: { currentPassword, newPassword }
C. Teacher Portal Routes (All require Teacher JWT)
GET  /api/teacher/my-assignments
     → Return teacherAssignments for this staffId

GET  /api/teacher/content
     → Return teacherContent for this staffId

POST /api/teacher/content
     Body: { title, description, contentType, className, section, subject, fileData, fileName }
     → Save to teacherContent collection
     → fileData: accept base64 string (frontend will convert file to base64 before sending)

PATCH /api/teacher/content/:id/toggle-publish
     → Flip isPublished true/false

DELETE /api/teacher/content/:id
     → Delete content item

GET  /api/teacher/quizzes
     → Return all quizzes for this staffId

POST /api/teacher/quizzes
     Body: full quiz object with questions array
     → Save to teacherQuizzes

PUT  /api/teacher/quizzes/:id
     → Update quiz (only if no attempts yet)

DELETE /api/teacher/quizzes/:id
     → Delete quiz (only if no attempts yet)

PATCH /api/teacher/quizzes/:id/toggle-publish
     → Flip isPublished

GET  /api/teacher/quizzes/:id/attempts
     → Return all quizAttempts for this quizId
     → Include student names

PATCH /api/teacher/quizzes/:id/attempts/:attemptId/grade-short
     Body: { questionIndex, marksAwarded }
     → Manually grade a short answer question
     → Recalculate total, percentage, grade, isPassed

D. Student Portal Routes (All require Student JWT)
POST /api/student-portal/login
     Body: { studentId, password }
     → Find in studentPortalAccounts
     → Compare bcrypt password
     → If isFirstLogin === true, include flag in response so frontend shows change-password screen
     → JWT payload: { studentId, studentName, className, section, role: "student" }

POST /api/student-portal/change-password
     Auth: Student JWT
     Body: { currentPassword, newPassword }
     → After success, set isFirstLogin = false

GET  /api/student-portal/dashboard
     Auth: Student JWT
     → Return:
         - Student profile from `students` collection
         - Active quizzes count (published, time slot currently active, not yet attempted)
         - Completed quizzes count
         - Pending fees total (from `feevouchers` collection where studentId matches)
         - This month attendance % (from `attendancerecords` collection)

GET  /api/student-portal/content
     Auth: Student JWT
     → Return teacherContent where className+section match AND isPublished = true
     → Group by subject in response

GET  /api/student-portal/quizzes
     Auth: Student JWT
     → Return teacherQuizzes where className+section match AND isPublished = true
     → For each quiz add:
         status: "upcoming" | "active" | "expired"  (compare current time vs startDateTime/endDateTime)
         attempted: true/false  (check quizAttempts collection)
         myResult: { grade, percentage, isPassed } if attempted

GET  /api/student-portal/quizzes/:id/start
     Auth: Student JWT
     → Return quiz WITHOUT correctAnswer fields
     → Validate: isPublished=true, current time is within slot, student has NOT attempted yet
     → If already attempted: return 400 "Already submitted"
     → If outside time slot: return 400 with message

POST /api/student-portal/quizzes/:id/submit
     Auth: Student JWT
     Body: { answers: [{ questionIndex, givenAnswer }], timeTakenMinutes }
     → Validate not already submitted
     → Auto-grade MCQ and True/False by comparing with correctAnswer
     → Short answer: isCorrect = null, marksAwarded = 0 (pending teacher grading)
     → Calculate: totalMarksObtained, percentage, grade (A+≥90, A≥80, B≥70, C≥60, D≥50, F<50)
     → isPassed = totalMarksObtained >= quiz.passingMarks
     → Save to quizAttempts, return result immediately

GET  /api/student-portal/results
     Auth: Student JWT
     → Return all quizAttempts for this studentId
     → Join with quiz title, subject, teacher name

GET  /api/student-portal/fees
     Auth: Student JWT
     → Query existing `feevouchers` collection where studentId = this student's ID
     → Return: each voucher month, total amount, paid amount, due date, status

GET  /api/student-portal/attendance
     Auth: Student JWT
     → Query existing `attendancerecords` for this student
     → Return month-by-month: presentDays, absentDays, percentage

PART 4 — IMPORTANT IMPLEMENTATION NOTES

File Storage: Convert PDF and image files to base64 string on the frontend using FileReader API before sending to backend. Store as base64 string in MongoDB. For display, use <iframe src="data:application/pdf;base64,..." for PDFs and <img src="data:image/jpeg;base64,..." for images. Do not use any external file storage service.
JWT Middleware: Create separate middleware functions:

requireCurriculumAdmin — checks existing curriculum token
requireTeacher — checks teacher JWT, attaches staffId to req
requireStudent — checks student JWT, attaches studentId, className, section to req
Each middleware checks role field in JWT payload.


Grade Calculation Function (reuse across codebase):

   A+ = percentage >= 90
   A  = percentage >= 80
   B  = percentage >= 70
   C  = percentage >= 60
   D  = percentage >= 50
   F  = percentage < 50

Student ID Lookup for Fees and Attendance: The student's studentId field in studentPortalAccounts (e.g., C5-A-2026-0007) is the same studentId field used in the existing feevouchers and attendancerecords collections. Use it directly for cross-collection queries.
Teacher Default Password: When a teacher logs in for the first time, their default password is their staffId value from the staff collection (e.g., STF-2024-001). Store this as bcrypt hash in a new teacherAuthPasswords collection with fields { staffId, passwordHash, updatedAt }. On first teacher login attempt, if no record exists in this collection, auto-create one with the default password.
Time Slot Enforcement: On the backend /api/student-portal/quizzes/:id/start, always check new Date() >= quiz.startDateTime && new Date() <= quiz.endDateTime before returning quiz questions. Never trust the frontend for this check.
Prevent Double Submission: Before saving a quiz attempt, always query quizAttempts collection for existing { quizId, studentId } record. If found, return 400 error.
Routing: Add all new routes to the existing Wouter router in the frontend. Teacher routes under /curriculum/teacher/... and student routes under /curriculum/student/.... The existing curriculum admin routes remain unchanged under /curriculum/....
Sidebar: Teacher portal and Student portal each have their own sidebar component. They do NOT use the existing curriculum admin sidebar.
Session Separation: Three different localStorage keys for three different sessions:

curriculum_token — existing admin (unchanged)
teacher_token — new teacher JWT
student_token — new student JWT
